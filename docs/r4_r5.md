## 2. New methods

### 2.1 Categorical Generalization

| Method | Description |
|---|---|
| `generalize_categorical_column(column, mapping, default=None)` | Replaces categorical values with broader categories using a user-defined mapping. Column must be declared `"categorical"` in metadata. |

### 2.2 Laplace Perturbation

| Method | Description |
|---|---|
| `perturb_numeric_column_laplace(column, sensitivity=1.0, epsilon=1.0, random_state=None)` | Adds Laplace noise to a numeric column following the differential privacy model. Column must be declared `"numeric"` in metadata. |

### 2.3 HMAC Pseudonymisation

| Method | Description |
|---|---|
| `deidentify_with_hmac_pseudonyms(identifier_column=None, secret_key=None)` | Replaces identifier values with reversible HMAC-SHA256 pseudonyms. Mappings are stored for reversal. |
| `reverse_hmac_pseudonyms(identifier_column=None)` | Restores original values for columns previously pseudonymized with `deidentify_with_hmac_pseudonyms`. |

## 3. Implementation

### 3.1 Categorical Generalization

**Role:** Reduces the granularity of categorical attributes by mapping specific values to broader categories, limiting the information an attacker can extract.

**Location:** `src/dataset_anonymization/manager.py`

**Main operations:**
- Verifies the column is declared `"categorical"` in metadata
- Applies the user-defined mapping using `pandas.Series.map()`
- Unmatched values are kept as-is or replaced by `default` if provided

---

### 3.2 Laplace Perturbation

**Role:** Adds calibrated random noise to numeric columns following the differential privacy model, providing formal mathematical privacy guarantees.

**Location:** `src/dataset_anonymization/manager.py`

**Main operations:**
- Computes the noise scale as `sensitivity / epsilon`
- Generates Laplace noise using `numpy.random.Generator.laplace()`
- Clips perturbed values to the original column range
- Integer columns stay integer (rounded), float columns stay float

---

### 3.3 HMAC Pseudonymisation

**Role:** Replaces identifiers with deterministic, reversible pseudonyms using HMAC-SHA256. Implements the keyed-hash pseudonymisation technique recommended by ENISA (2019).

**Location:** `src/dataset_anonymization/manager.py`

**Main operations:**
- Generates a random 32-byte secret key automatically if not provided
- For each unique value: `token = HMAC-SHA256(key, value)[:16]`
- Pseudonym format: `HMAC-<token>`
- The mapping is stored in `_hmac_mappings` for later reversal
- `reverse_hmac_pseudonyms()` inverts the mapping and removes it after use
